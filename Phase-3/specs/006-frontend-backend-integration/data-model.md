# Data Model: Task Entity

## Task Object

The Task entity represents a user's task in the system.

### Fields

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| id | string (UUID) | Yes | Unique identifier for the task |
| title | string | Yes | The task title/description |
| description | string | No | Additional details about the task |
| completed | boolean | No | Whether the task is completed (defaults to false) |
| createdAt | string (ISO date) | Yes | When the task was created |
| updatedAt | string (ISO date) | Yes | When the task was last updated |

### Validation Rules

- **Title**: Must be non-empty string, maximum 255 characters
- **Description**: Optional, maximum 1000 characters
- **Completed**: Boolean value, defaults to false
- **ID**: Generated by backend, follows UUID format

### State Transitions

- **Pending** → **Completed**: When user marks task as complete
- **Completed** → **Pending**: When user unmarks task as complete

## API Operations

### Create Task
- **Endpoint**: `POST /api/tasks`
- **Request Body**:
  ```json
  {
    "title": "string",
    "description": "string?"
  }
  ```
- **Response**:
  ```json
  {
    "id": "uuid",
    "title": "string",
    "completed": false
  }
  ```

### Read Tasks
- **Endpoint**: `GET /api/tasks`
- **Response**:
  ```json
  {
    "data": [
      {
        "id": "uuid",
        "title": "string",
        "description": "string?",
        "completed": boolean,
        "createdAt": "ISO date",
        "updatedAt": "ISO date"
      }
    ]
  }
  ```

### Update Task
- **Endpoint**: `PUT /PATCH /api/tasks/:id`
- **Request Body**:
  ```json
  {
    "title": "string?",
    "description": "string?",
    "completed": boolean?
  }
  ```

### Delete Task
- **Endpoint**: `DELETE /api/tasks/:id`

## Frontend Representation

The frontend represents tasks using the same structure as the backend response, with additional UI state as needed:

```typescript
interface Task {
  id: string;
  title: string;
  description?: string;
  completed: boolean;
  createdAt: string;
  updatedAt: string;
}
```

## Authentication Requirements

All task operations require a valid JWT token in the Authorization header:
```
Authorization: Bearer <token>
```

The API client automatically attaches this token via interceptors.
